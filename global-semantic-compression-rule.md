# Global Semantic Compression Rule

The live corpus preserves semantics, not documents.

For every document:

1. identify all currently useful semantic content;
2. if that content is already represented elsewhere, delete the document;
3. if only part remains useful, extract and compress it into the proper living owner;
4. verify that no current semantic capability was lost;
5. delete the obsolete container.

Historical record, development sequence, migration provenance, former importance, and isolated useful fragments are not sufficient reasons for retention. Git history is the archive.

The default disposition of a fully superseded document is deletion. The default disposition of a mostly superseded document is extraction, compression, integration, verification, and deletion.

Retention is justified only when the document's independent structure performs a substantial current semantic, proof, frontier, control, or navigation function that cannot be provided more simply elsewhere.

The global archive/delete pass terminates only when no further document can be removed through semantic absorption and compression without loss of current rigor, clarity, ownership, frontier visibility, or recoverability.
