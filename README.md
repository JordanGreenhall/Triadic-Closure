## Corpus architecture

The living reader path has five levels:

1. foundational architectonics and generic domain method;
2. mathematical construction and instruments;
3. physics, from domain entry through spacetime, matter, mass, gravity, and the Standard Model boundary;
4. the physics–chemistry interface;
5. chemistry, from the domain gate through bonding, geometry, recursion, aromaticity, reactions, kinetics, and thermodynamics.

Control documents govern authority, standing, execution, and frontiers. Supporting pages remain only when they provide substantial current semantic or derivational value not more cleanly absorbed by a canonical owner.

## Authority

Authority is claim-specific, not file-wide. Apply the order in [Corpus Lineage](corpus-lineage.md): exact direct ruling until propagation; surviving repair or adjudication; integrated ledger; repaired complete derivation; unsuperseded earlier derivation; scoped frontier; necessary downstream expenditure; summary; compiler inventory. Chronology alone never settles authority.

## Standing and registration

Claims offered for warrant are Open, Conjectured, or Secured. A pure posit is a premise category with no warrant claimed. Inherited names are Native, Unregistered, Registered, or Registered–Sealed. Registration licenses mapped semantic content; it is not proof strength. `Locked actual` is deprecated. See [Claim Status Vocabulary](claim-status-vocabulary.md).

## Normalization and semantic compression

Normalization activates exactly one semantic unit or one expressly global pass. Neighboring theory remains read-only except for required propagation. Do not invent theory, average contradictions, infer authority from recency, or claim completion before real verification, fresh-reader review, and a second no-change cycle.

The live corpus preserves semantics, not documents. A fully superseded document is deleted. A partly useful document is decomposed: surviving useful material is compressed into its proper living owner, verified, and the obsolete container is deleted. Historical record, development history, migration provenance, and isolated useful fragments are not sufficient reasons for retention because Git history already preserves prior states.

A document remains only when its independent structure performs a substantial current semantic, proof, frontier, control, or navigation function that cannot be provided more simply elsewhere.
